## Domains

The `vercel domains` command is used to manage domains under an account, providing functionality to list, inspect, add, remove, purchase, move, transfer-in, and verify domains.

<Note type="warning">
  This command is for <b>advanced use only</b>. You can manage domains with
  further options and greater control under a project's Domains tab from the{' '}
  <OrgName /> Dashboard.
</Note>

### Basic Usage

<Snippet width="100%"dark text="vercel domains ls" />
<Caption>Using the <InlineCode>vercel domains</InlineCode> command to list all domains under an account.</Caption>

### Extended Usage

<Snippet width="100%"dark text="vercel domains inspect [domain]" />
<Caption>Using the <InlineCode>vercel domains</InlineCode> command to retrieve information about a specific domain.</Caption>

<Snippet width="100%"dark text="vercel domains add [domain]" />
<Caption>Using the <InlineCode>vercel domains</InlineCode> command to add a domain to an account.</Caption>

<Snippet width="100%"dark text="vercel domains rm [domain]" />
<Caption>Using the <InlineCode>vercel domains</InlineCode> command to remove a domain from an account.</Caption>

<Snippet width="100%"dark text="vercel domains buy [domain]" />
<Caption>Using the <InlineCode>vercel domains</InlineCode> command to buy a domain for an account.</Caption>

<Snippet width="100%"dark text="vercel domains move [domain] [account-name]" />
<Caption>Using the <InlineCode>vercel domains</InlineCode> command to move a domain to another account.</Caption>

<Snippet width="100%"dark text="vercel domains transfer-in [domain]" />
<Caption>Using the <InlineCode>vercel domains</InlineCode> command to transfer in a domain to an account.</Caption>

<Snippet width="100%"dark text="vercel domains verify [domain]" />
<Caption>Using the <InlineCode>vercel domains</InlineCode> command to verify a domain for an account.</Caption>

### Global Options

The following [global options](#options) can be passed when using the `vercel domains` command:

- `--debug`
- `--global-config`
- `--help`
- `--local-config`
- `--scope`
- `--token`

For more information on global options and their usage, refer to the [options section](#options).
